# Project Structure

> OSS repository layout for NanthAI Edge: Convex backend plus React/Vite web client.

```
nanthai-edge-oss/
├── convex/                         # Convex backend functions, schema, tools, tests
│   ├── schema.ts                    # Database schema entry point
│   ├── schema_tables_core.ts        # Chats, messages, documents, files, search state
│   ├── schema_tables_catalog.ts     # Models, skills, memory/query catalogs
│   ├── schema_tables_user.ts        # Preferences, jobs, secrets, entitlements, tokens
│   ├── schema_tables_runtime.ts     # Runtime workspace and sandbox state
│   ├── schema_validators.ts         # Shared Convex validators
│   ├── auth.config.ts               # Clerk JWT auth provider
│   ├── crons.ts                     # Scheduled maintenance and sync jobs
│   ├── http.ts                      # Convex HTTP routes such as downloads/webhooks
│   ├── account/                     # Account-level actions and mutations
│   ├── autonomous/                  # Autonomous/group discussion runs
│   ├── capabilities/                # Capability and entitlement queries/mutations
│   ├── chat/                        # Chat lifecycle, streaming, audio, retry, branching
│   ├── documents/                   # Canonical document/version mutations and queries
│   ├── drive_picker/                # Google Drive Picker ingestion helpers
│   ├── favorites/                   # Favorite prompts/items
│   ├── folders/                     # Chat folder management
│   ├── integrations/                # Integration request gates and shared checks
│   ├── jobs/                        # Cleanup helpers
│   ├── knowledge_base/              # Knowledge Base import/list/query flows
│   ├── lib/                         # Auth, OpenRouter, rate limits, crypto, shared helpers
│   ├── memory/                      # Memory extraction, retrieval, embeddings, import flow
│   ├── models/                      # OpenRouter catalog sync, filters, guidance scoring
│   ├── nodePositions/               # Ideascape node position persistence
│   ├── oauth/                       # OAuth and credential flows for integrations
│   ├── participants/                # Chat participant management
```

## Generated and Local Files

- `convex/_generated/` is generated by `npx convex dev` and is intentionally not committed.
- Root `.env.local` is local Convex CLI/backend configuration copied from `.env.example`.
- `web/.env.local` is local Vite frontend configuration copied from `web/.env.example`.
- `node_modules/` directories are local install output and are not committed.

## Common Entry Points

| Task | Start Here |
|------|------------|
| Add/change database tables | `convex/schema.ts` and `convex/schema_tables_*.ts` |
| Change chat send/stream behavior | `convex/chat/` |
| Add an AI tool | `convex/tools/registry.ts`, `convex/tools/index.ts`, and the relevant provider/tool module |
| Change skills | `convex/skills/` and `convex/skills/catalog/` |
| Change OpenRouter request behavior | `convex/lib/openrouter*.ts` and `convex/chat/actions_run_generation_*` |
| Change model picker/catalog behavior | `convex/models/` and `web/src` picker/settings surfaces |
| Change Knowledge Base behavior | `convex/knowledge_base/`, `convex/documents/`, and `web/src/routes/KnowledgeBasePage.tsx` |
| Change web chat UI | `web/src/routes/ChatPage.tsx`, `web/src/components/chat/`, and related hooks |
| Change public pages | `web/src/pages/` and `web/src/components/edge-site/` |
| Change auth/onboarding | `web/src/routes/AuthGuard.tsx`, `SignInPage.tsx`, `OnboardingPage.tsx`, and Clerk env config |
| Change integration OAuth | `convex/oauth/`, `convex/tools/<provider>/`, and `web/src/lib/providerOAuth.ts` |
